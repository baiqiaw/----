## 3. 开发准备工作

### 3.1 环境搭建

按照《开发环境搭建指南》完成以下准备工作：

1. **安装开发工具**：
   - Node.js 20 LTS+
   - pnpm（包管理器）
   - PostgreSQL 14+
   - Redis 7+
   - Docker（可选，推荐）

2. **配置开发环境**：
   - 配置IDE（VS Code推荐）
   - 安装必要的扩展（ESLint、Prettier等）
   - 配置Git和代码仓库

3. **初始化项目**：
   - 创建项目目录结构
   - 初始化各子项目（backend、web、desktop）
   - 配置环境变量

**详细步骤**：参考 [开发环境搭建指南](./06-部署文档/开发环境搭建指南.md)

### 3.2 代码仓库搭建

按照开发规范文档，需要完成：

1. **创建Git仓库**：
   - 初始化Git仓库
   - 配置分支策略（main、develop、feature/*等）
   - 设置代码审查流程

2. **配置CI/CD**：
   - 配置GitHub Actions或GitLab CI
   - 设置自动化测试
   - 配置自动化构建

**详细步骤**：参考 [开发环境搭建指南](./06-部署文档/开发环境搭建指南.md) 第3节

### 3.3 数据库初始化

按照数据库设计文档，需要完成：

1. **创建数据库**：
   - 创建PostgreSQL数据库
   - 运行数据库迁移脚本

2. **初始化数据**：
   - 初始化字典表数据
   - 创建初始管理员账户

**详细步骤**：参考 [数据库设计文档](./02-设计文档/数据库设计文档.md)

### 5.1 代码规范

所有代码必须遵循《开发规范文档》：

- **代码风格**：遵循ESLint和Prettier配置
- **命名规范**：遵循统一的命名规范
- **提交规范**：遵循Conventional Commits规范
- **代码审查**：所有代码必须经过审查

**参考文档**：[开发规范文档](./02-设计文档/开发规范文档.md)

- **单元测试**：覆盖率≥80%

## 7. 常见问题

### 7.1 如何开始开发？

1. 阅读《开发环境搭建指南》，完成环境搭建
2. 阅读相关模块的详细设计文档
3. 按照开发顺序建议，从第一阶段开始
4. 遵循开发规范，编写代码

### 7.2 遇到设计问题怎么办？

1. 查阅相关设计文档
2. 查阅API接口详细设计文档
3. 查阅数据库设计文档
4. 如文档未覆盖，提出设计问题，更新文档

### 7.3 如何提交代码？

1. 遵循Conventional Commits规范
2. 确保代码通过ESLint检查
3. 确保单元测试通过
4. 创建Pull Request，等待代码审查
